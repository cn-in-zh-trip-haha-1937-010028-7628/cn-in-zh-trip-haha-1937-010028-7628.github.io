Пошаговая настройка
1

Заказываешь VPS на ps.kz или hoster.kz

При заказе выбираешь:
— ОС: Windows Server 2019/2022 (устанавливать самому не нужно — хостер ставит автоматически)
— минимум 2 CPU, 2GB RAM, 50GB SSD
— регион: Казахстан
После оплаты на почту придёт IP адрес, логин и пароль от сервера.

2

Подключаешься к VPS по RDP (ещё из Казахстана)

На Windows: нажми Win+R → mstsc → Enter
Вводишь IP сервера, логин и пароль которые прислал хостер. Попал на рабочий стол — всё работает.

// Win+R, потом:
mstsc
// Вводишь IP: например 185.xxx.xxx.xxx
// Логин: Administrator
// Пароль: (из письма хостера)
3

Меняешь RDP порт с 3389 на нестандартный

Это важно — стандартный порт 3389 Китай иногда блокирует. Делаешь прямо на VPS через реестр:

// На VPS открываешь: Win+R → regedit
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\
Control\Terminal Server\WinStations\RDP-Tcp

// Находишь параметр PortNumber
// Меняешь с 3389 на например 8443
// Перезагружаешь сервер
ℹ После смены порта подключаться надо так: 185.xxx.xxx.xxx:8443
4

Открываешь порт в брандмауэре Windows на VPS

// На VPS открываешь PowerShell от имени администратора:
New-NetFirewallRule -DisplayName "RDP Custom" `
  -Direction Inbound -Protocol TCP `
  -LocalPort 8443 -Action Allow
5

Проверяешь что всё работает

Открываешь mstsc, вводишь IP:8443 — должен подключиться. Попал на рабочий стол VPS — отлично.

6

С VPS подключаешься к своей рабочей удалёнке

Уже находясь на рабочем столе VPS — открываешь mstsc ещё раз и вводишь IP/адрес своей рабочей удалёнки. Это и есть двойной RDP.

7

Сообщаешь айтишникам новый IP

Узнаёшь IP своего VPS (просто гуглишь "my ip" с VPS) и даёшь его айтишникам. Просишь добавить в whitelist.

FAQ — Нестандартные ситуации
Не могу подключиться к VPS из Китая ▶
1. Проверь интернет — открывается ли вообще что-то?
2. Попробуй другой порт — некоторые провайдеры в Китае блокируют определённые порты. Попробуй 443, 80, 8080.
3. Смени порт RDP на сервере на 443 — этот порт никогда не блокируется (HTTPS).
4. Попробуй с мобильного интернета вместо Wi-Fi — иногда университетский Wi-Fi блокирует больше.
VPS завис, не отвечает ▶
Заходишь в панель управления хостера (ps.kz или hoster.kz) через браузер → находишь свой сервер → нажимаешь Перезагрузить. Через 2–3 минуты снова подключаешься.
Забыл пароль от VPS ▶
В панели управления хостера есть функция Сбросить пароль или Консоль — через неё можно войти без пароля и сменить его.
Медленно работает, лагает ▶
1. В настройках RDP снизь качество: Connection → Experience → выбери "Modem (56 kbps)".
2. Отключи фоновые рисунки и анимации на VPS.
3. Закрой лишние программы на VPS.
4. Если проблема постоянная — апгрейд тарифа VPS до 4GB RAM.
Айтишники говорят что не могут добавить IP в whitelist ▶
Попроси у хостера статический IP — он не меняется. Доплата ~500–1000 KZT/мес. Динамический IP может меняться при перезагрузке сервера, тогда придётся каждый раз обновлять у айтишников.
Сервер взломали / подозрительная активность ▶
1. Сразу смени пароль от VPS на сложный (20+ символов).
2. Ограничь доступ по IP — разреши RDP только со своего IP (в брандмауэре Windows).
3. Проверь логи входов: Просмотр событий → Безопасность → Event ID 4625 (неудачные входы).
Нужно скачать файл с рабочего компа на свой ▶
В настройках RDP при подключении к рабочей удалёнке включи Local Resources → Drives — тогда твои диски будут видны на рабочем компе и можно просто скопировать файлы.
Хостер перестал работать или закрылся ▶
Сделай резервный аккаунт на втором хостере (если на ps.kz — зарегись ещё и на hoster.kz). При проблемах быстро поднимешь новый VPS за 15 минут. Данные рабочей удалёнки хранятся на серверах работодателя, VPS только промежуточная точка.
