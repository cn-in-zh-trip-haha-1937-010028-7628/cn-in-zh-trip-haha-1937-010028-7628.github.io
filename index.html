Пошаговая настройка
1

Заказываешь VPS на ps.kz или hoster.kz

При заказе выбираешь:
— ОС: Windows Server 2019/2022 (устанавливать самому не нужно — хостер ставит автоматически)
— минимум 2 CPU, 2GB RAM, 50GB SSD
— регион: Казахстан
После оплаты на почту придёт IP адрес, логин и пароль от сервера.

2

Подключаешься к VPS по RDP (ещё из Казахстана)

Нажми Win+R → mstsc → Enter
Вводишь IP сервера, логин и пароль которые прислал хостер.

// Win+R, потом:
mstsc
// Вводишь IP: например 185.xxx.xxx.xxx
// Логин: Administrator
// Пароль: (из письма хостера)

Открываешь Microsoft Remote Desktop из App Store.
Нажимаешь + → Add PC, вводишь IP сервера.
Логин: Administrator, пароль из письма хостера.

# Или через встроенный браузер (менее удобно):
# App Store → Microsoft Remote Desktop → Add PC
PC name: 185.xxx.xxx.xxx
User account: Administrator

Открываешь Remmina, нажимаешь + (новое соединение).
Protocol: RDP, Server: IP сервера, Username: Administrator.

# Или через терминал напрямую:
xfreerdp /v:185.xxx.xxx.xxx /u:Administrator /p:ПАРОЛЬ
3

Меняешь RDP порт с 3389 на нестандартный (делается на VPS)

Это важно — стандартный порт 3389 Китай иногда блокирует. Делаешь прямо на VPS (на его рабочем столе):

// На VPS: Win+R → regedit
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp

// Находишь PortNumber → меняешь с 3389 на 8443
// Перезагружаешь сервер
ℹПосле смены порта подключаться: 185.xxx.xxx.xxx:8443
4

Открываешь порт в брандмауэре Windows на VPS

// На VPS → PowerShell от имени администратора:
New-NetFirewallRule -DisplayName "RDP Custom" `
  -Direction Inbound -Protocol TCP `
  -LocalPort 8443 -Action Allow
5

Проверяешь подключение с новым портом

Открываешь mstsc, вводишь IP:8443 — должен подключиться.

В Microsoft Remote Desktop меняешь PC name на IP:8443 → подключаешься.

xfreerdp /v:185.xxx.xxx.xxx:8443 /u:Administrator /p:ПАРОЛЬ

Или в Remmina — меняешь Server на IP:8443.

6

С VPS подключаешься к своей рабочей удалёнке

Уже находясь на рабочем столе VPS — открываешь mstsc ещё раз и вводишь IP/адрес своей рабочей удалёнки. Это и есть двойной RDP.

7

Сообщаешь айтишникам новый IP

Узнаёшь IP своего VPS (гуглишь "my ip" прямо с VPS) и даёшь его айтишникам. Просишь добавить в whitelist.

FAQ — Нестандартные ситуации
Не могу подключиться к VPS из Китая▶
1. Проверь интернет — открывается ли вообще что-то?
2. Попробуй другой порт — некоторые провайдеры блокируют порты. Попробуй 443, 80, 8080.
3. Смени порт RDP на сервере на 443 — этот порт никогда не блокируется (HTTPS).
4. Попробуй с мобильного интернета вместо Wi-Fi — университетский Wi-Fi иногда блокирует больше.
Mac: Убедись что в Microsoft Remote Desktop правильно указан порт: IP:8443 в поле PC name.
Linux: Проверь что freerdp установлен: xfreerdp --version. Если нет — sudo pacman -S freerdp.
VPS завис, не отвечает▶
Заходишь в панель управления хостера (ps.kz или hoster.kz) через браузер → находишь свой сервер → нажимаешь Перезагрузить. Через 2–3 минуты снова подключаешься.
Забыл пароль от VPS▶
В панели управления хостера есть функция Сбросить пароль или Консоль — через неё можно войти без пароля и сменить его.
Медленно работает, лагает▶
Windows: В mstsc → Options → Experience → выбери "Modem (56 kbps)".
Mac: В Microsoft Remote Desktop → Edit PC → Display → сними галочки с "Use all monitors" и "Start session in full screen". В качестве Color quality выбери меньшее значение.
Linux: Добавь флаги: xfreerdp /v:IP:8443 /u:Admin /p:PASS /bpp:16 /compression

Также: отключи фоновые рисунки и анимации на VPS. Если постоянно лагает — апгрейд тарифа до 4GB RAM.
Айтишники не могут добавить IP в whitelist▶
Попроси у хостера статический IP — он не меняется. Доплата ~500–1000 KZT/мес. Динамический IP может меняться при перезагрузке сервера.
Сервер взломали / подозрительная активность▶
1. Сразу смени пароль от VPS на сложный (20+ символов).
2. Ограничь доступ по IP — разреши RDP только со своего IP (в брандмауэре Windows на VPS).
3. Проверь логи: Просмотр событий → Безопасность → Event ID 4625 (неудачные входы).
Нужно скачать файл с рабочего компа на свой▶
В настройках mstsc → Local Resources → Drives — включи свои диски. Они появятся на рабочем компе и можно скопировать файлы.
В Microsoft Remote Desktop → Edit PC → Folders → добавь папку на маке. Она появится на VPS как сетевой диск.
Добавь флаг: xfreerdp /drive:home,/home/USERNAME — твоя домашняя папка появится на VPS.
Хостер перестал работать или закрылся▶
Сделай резервный аккаунт на втором хостере (если на ps.kz — зарегись ещё и на hoster.kz). При проблемах быстро поднимешь новый VPS за 15 минут. Данные рабочей удалёнки хранятся у работодателя — VPS только промежуточная точка.
Как сохранить подключение чтобы не вводить каждый раз Как сохранить подключение чтобы не вводить каждый раз Как сохранить подключение чтобы не вводить каждый раз ▶
В mstsc → Save As — сохраняешь файл .rdp на рабочий стол. Двойной клик — и сразу подключается.
В Microsoft Remote Desktop подключение сохраняется автоматически после добавления. Просто кликаешь по нему.
В Remmina сохраняешь профиль подключения. Или создаёшь alias в .bashrc:
alias vps='xfreerdp /v:IP:8443 /u:Administrator /p:ПАРОЛЬ'
Потом просто пишешь vps в терминале.
